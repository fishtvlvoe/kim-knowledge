# BWPA 風險行動模型

Best / Worst / Possible / Actual 四維度決策框架。Kim 黃晨皓用來分析任何重大決策的標準工具，特別適合面對不確定性時理清思路。

## 四個維度

### Best（最好的情況）

如果一切順利，最好的結果是什麼？如何讓它發生？

這一步不是樂觀主義，而是定義**成功的天花板**，以及**通向天花板的具體路徑**。

問：
- 最好的情況能到什麼程度？
- 需要哪幾個條件同時成立，才能實現最好的情況？
- 我能主動創造哪些條件？

### Worst（最壞的情況）

如果全部出錯，最壞的情況是什麼？如何預防？能不能承受？

這一步是**底線確認**。能接受最壞的情況，才能無後顧之憂地行動。

問：
- 最壞的情況是什麼？能不能接受？
- 哪一種失敗是無法承受的（紅線）？
- 我能提前做哪些預防措施？

### Possible（最可能的情況）

現實中，最有可能發生的情況是什麼？

這是決策的**基準線**，通常介於 Best 和 Worst 之間。基於過去的數據、市場的現實、人性的規律來推算。

問：
- 類似情況通常的結果是什麼？
- 這個機率估計的依據是什麼？
- 有哪些變量會讓結果偏向 Best 或 Worst？

### Actual（目前真實的情況）

現在實際是什麼狀況？

這是所有分析的起點。許多決策困難，是因為當事人對「目前真實情況」有誤判或迴避。

問：
- 現在真實的數字是什麼？（資源、時間、能力、關係）
- 我對現況的哪些評估是樂觀偏差？
- 跟利害關係人說清楚現況，他們的反應是什麼？

## 應用 SOP

1. 先定義決策：「我在考慮要不要做 X」
2. 找出積木：「這個決策的關鍵變量是什麼？（A × B × C）」
3. 四維分析：逐一回答 Best/Worst/Possible/Actual
4. **關鍵問**：「最壞的情況，我能不能接受？」
   - 能接受 → 行動
   - 不能接受 → 重新設計方案，降低 Worst 的影響
5. 積極語言收尾：把「我做不到 X」轉成「我能如何做到 X？」

## 範例應用

**場景：要不要開一個線上課程平台**

| 維度 | 分析 |
|------|------|
| Best | 平台月收 100 萬，核心講師 10 位，學員口碑形成飛輪 |
| Worst | 6 個月沒有學員，技術出問題，講師撤出 |
| Possible | 第一年月收 10-20 萬，靠 3-5 位核心講師維持 |
| Actual | 目前 0 學員、0 技術底子、有 2 位講師願意嘗試 |

**決策重點**：Worst（6 個月沒學員）能接受嗎？如果資金撐得過 6 個月，就可以行動。

## 四個原則

1. **誠實面對 Actual**：樂觀偏差是最常見的失誤來源
2. **接受才行動**：不能接受 Worst，就先降低 Worst，再做決定
3. **Possible 靠數據**：不能靠直覺，要靠可類比的過去案例
4. **積極語言**：分析完之後，把所有消極描述轉成「我能如何…？」的問句

## 觸發問題

- 最壞的情況是什麼？我能不能接受？
- 目前真實的情況，我有沒有用數字說清楚？
- 如果要讓 Best 發生，最關鍵的 1 個動作是什麼？
