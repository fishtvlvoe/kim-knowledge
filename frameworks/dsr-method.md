# DSR 三構法

解構（Deconstruct）→ 架構（Structure）→ 重構（Reconstruct）。Kim 黃晨皓用來突破認知框架、創造創新解法的三步驟方法。

## 三個步驟

### 解構（Deconstruct）

打碎現有的認知框架，找出藏在裡面的假設。

核心問題：「現有的理解有哪些假設？哪一個假設最值得挑戰？」

不挑戰假設，就只是在舊框架裡打轉。解構的目的不是破壞，是**釋放積木**，讓它們可以用新的方式組合。

### 架構（Structure）

基於解構後的洞察，建立更全面、更精確的模型。

核心問題：「如果舊假設不成立，正確的結構應該是什麼？」

這一步是重建秩序。把釋放出來的積木，加上新的認識，重新排列成更有解釋力的框架。

### 重構（Reconstruct）

把「已知 A」與「已知 B」組合出新的未知 C。

核心問題：「把這兩個已知重新組合，會創造什麼以前沒有的東西？」

創新往往不需要全新的原料，只需要**舊積木的新排列**。

## 豬肉比喻

- **豬肉** = 原始積木（已知 A）
- **絞肉** = 解構 + 架構（把豬肉重新理解成「可重組的蛋白質纖維」）
- **香腸** = 重構的創新產品（已知 A × 已知 B → 新的未知 C）

如果你沒有解構「豬肉只能烤來吃」的假設，你就不會想到把它絞碎再塑形成香腸。

另一個比喻：**書架 + 門 = 秘密通道**。書架是已知的，門是已知的，但這個組合能創造全新的體驗。

## 應用步驟

1. **指出現有假設**：「這件事大家通常認為是 X，因為他們假設了 Y」
2. **挑戰最強假設**：「如果 Y 不成立，X 還對嗎？」
3. **用新眼光重建**：「去掉 Y 的前提，這件事的正確結構是什麼？」
4. **找重組機會**：「現有的積木如何以新方式排列，創造以前沒有的東西？」

## 與積木思維的關係

DSR 是積木思維的動態版本：

```
積木思維 = 如何看（靜態）
DSR 三構法 = 如何突破（動態）
```

積木思維讓你看清系統的組成，DSR 三構法讓你知道如何改造它。

## 觸發問題

- 這件事大家「理所當然」的假設是什麼？
- 如果把最核心的假設去掉，整件事會怎麼變？
- 哪兩個「已知」組合在一起，會創造一個以前不存在的東西？
