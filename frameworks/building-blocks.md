# 積木思維

把任何複雜的事物拆解成最小單位，再透過重新組合創造全新的東西。Kim 黃晨皓的核心認識論基礎。

## 定義

**積木** = 構成事物的最小不可再分單位

任何系統、商業模式、知識框架、溝通行為，都可以被分解成幾個基本積木。找到積木，就找到了理解和創新的起點。

> 「通過將『已知』與『已知』的組合，變成一個全新的未知。」

## 組合規則

**結構 = 積木 A + 積木 B + … + 積木 N**

積木的威力不在單個積木，而在於**組合方式**。同樣的積木，不同的排列會產生完全不同的結果。

類比：豬肉（原料積木）→ 絞肉（重組）→ 香腸（創新產品）

## 複雜度分層

| 層級 | 積木數量 | 範例 |
|------|---------|------|
| 簡單系統 | ≤ 3 個積木 | 個人品牌 = 專業 × 個性 × 平台 |
| 中等系統 | 4 個積木 | 課程平台 = 目標用戶 × 課程內容 × 獲客管道 × 變現方式 |
| 複雜系統 | > 4 個積木 | 需先找核心 3 個，再逐層展開 |

**原則：** 積木越少越好。找到最精簡的拆法，才是真正理解了這件事。

## 公式範例

**銷售轉換率**
```
轉換率 = (觸達數 × 信任感 × 時機點) / 阻力
```
→ 阻力越大，轉換率越低。要提升轉換率，可以同時拉高分子、降低分母。

**品牌溢價**
```
品牌溢價 = (差異化程度 × 信任積累) / 市場認知成本
```
→ 市場認知成本高（用戶需要大量時間才能理解你），溢價就難。

**學習速度**
```
學習速度 = (輸入質量 × 實戰反饋密度) / 認知負荷
```
→ 高輸入質量 + 高密度反饋，才能快速掌握新領域。

**課程設計**
```
課程效果 = (核心痛點命中率 × 框架可操作性 × 學員參與度) / 認知跨度
```

## 應用 SOP

1. **分解**：把問題說出來，拆成積木清單（A + B + C）
2. **確認**：跟對方確認「這是全部的積木嗎？有沒有漏掉什麼？」
3. **重組**：把積木按不同方式排列，看看哪種組合能產生新價值
4. **公式化**：用 `X = A × B / C` 的形式把結論寫出來，讓邏輯一目瞭然

## 觸發問題

- 這件事最小的組成單位是什麼？
- 如果只能保留 3 個積木，我會保留哪 3 個？
- 這兩個已知的積木，組合在一起會變成什麼？
