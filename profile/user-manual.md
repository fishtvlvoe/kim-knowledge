# 阿金使用說明書

Kim 黃晨皓（阿金）的自述。了解他的優勢、工作方式、以及如何與他有效合作。

## 核心優勢

### 1. 結構化思維

能快速將複雜問題拆解成清晰的積木，並找出最關鍵的槓桿點。

看到一個問題，他不是往前衝，而是先問「這件事的結構是什麼？」，然後用最少的力氣撬動最大的改變。

### 2. 框架建構能力

能把實際經驗抽象成可傳授的框架，讓他人也能複製洞察。

他不只是「知道怎麼做」，他知道「為什麼這樣做」，並且能把背後的原理說清楚、教明白。

### 3. 快速找到問題關鍵點

「發現問題的關鍵點，多從實際問題出發，建立方案後避免自己執行」。

他的強項在診斷和設計，不在執行。他能在 10 分鐘內找出一個月都解決不了的問題的核心，然後告訴你怎麼修。

### 4. 意圖理解與溝通設計

能快速讀懂對話背後的真實意圖，並設計有效的表達方式。

他不只聽你說什麼，他聽你為什麼這樣說。這讓他在溝通和教學上特別有穿透力。

## 優勢的反面

每個優勢都有它的影子。

| 優勢 | 可能的反面 |
|------|---------|
| 結構化思維 | 可能會讓對方覺得「被框架化」，反感「套公式」的感受 |
| 框架建構 | 框架高度抽象，初學者可能感到距離感 |
| 快速診斷 | 太快給答案，對方還沒準備好，反而排斥 |
| 意圖理解 | 讀懂意圖後直接切入，跳過了對方需要的「被傾聽」過程 |

## 如何與他有效互動

1. **直接說問題，不要鋪陳太多背景** — 他的強項在快速找核心，給他核心材料即可
2. **準備好被反問** — 他習慣用問題幫你思考，這不是質疑，是他的工作方式
3. **說出你真正想要的，而不是你以為你要的** — 他最擅長區分這兩者，節省彼此的時間
4. **如果他的框架讓你感覺被套公式，直接說** — 他可以用更具體的方式重新說明

## 充電方式

- 解決有挑戰性的問題，特別是跨領域的結構性問題
- 教學與分享：把知識傳出去，看到對方有頓悟
- 與高密度思考者的對話
- 在自由度高的環境中探索

## 耗電方式

- 執行細節、繁瑣的行政工作
- 在意義不明確的會議中重複相同的討論
- 被迫在框架尚未清晰時就要輸出結果

## 理想工作環境

- 自由度高、試錯空間大
- 不需要循規蹈矩，重視結果不重視過程
- 有真實問題需要解決，不是為了完成表格而工作
- 有清晰的目標和判斷標準

## 理想團隊成員

- 執行力強：能把 Kim 的診斷和設計落地
- 細節導向：補上 Kim 不擅長的執行層面
- 有問題意識：能主動帶著真實問題來討論，而不是等著被告知做什麼
- 能接受直接的溝通方式：Kim 說話直接，不是因為不在乎，而是因為尊重對方的時間

## 角色定位

Kim 在組織或合作關係中，最適合的角色是：

**診斷者 + 框架設計者**，而不是執行者。

他最有價值的地方是：找到問題的根本原因、設計解決的框架、讓別人能夠照著做。他不是用來「做事」的，是用來「想清楚怎麼做事」的。
